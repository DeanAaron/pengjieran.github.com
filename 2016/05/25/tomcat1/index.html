tomcat性能配置

tomcat性能配置包含Executor线程池和Connectors的配置，有些配置在老的版本中不支持，而有些配置在新版本中却被移除了，所以，现在的配置项都会参考官方文档。
所有的配置都是在server.xml文件中

Executor配置选项

简介

Executor 是一个可以在tomcat各个组件之间共享使用同一个线程池的配置项，历史中connector 每一个都要配置单独的线程池，但是，这个配置项可以允许你在不同的connector之间，甚至是不同组件之间共享相同的线程池。
想使用自定义的Executor必须要实现org.apache.catalina.Executor接口。
Executor配置项是Service的子配置项，connector也是。

通用属性

所有Executor接口的实现都支持如下属性：

属性 描述
className 实现类的类名，该接口确保可以通过name引用找到实现并加入容器的整个生命循环中，默认的值是org.apache.catalina.core.StandardThreadExecutor
name 在server.xml中可以找到该线程池的名称，必须且唯一的。

扩展实现属性

默认实现支持如下属性：

属性 描述
threadPriority int值，线程池中线程的优先级，默认值是5(Thread.NORM_PRIORITY常量的值)
daemon 线程是否是守护线程，默认值是true
namePrefix 每一个在executor中的线程名的前缀，每个线程的线程名是namePrefix+threadNumber，以保证线程池中的唯一性
maxThreads 线程池中活动线程的最大数量，默认值是200
minSpareThreads 始终存活的最小线程数，默认值是25
maxIdleTime 空闲线程的最大存活时间，默认值60000(1分钟)，空闲线程始终会小于等于minSpareThreads数量
maxQueueSize 排队的请求的最大数量，默认值是Integer.MAX_VALUE
prestartminSpareThreads boolean,线程池初始化时是否初始化minSpareThreads数量的线程，默认值false
threadRenewalDelay 如果配置了ThreadLocalLeakPreventionListener，当contexts关闭后，它会通知executor。在context关闭后，线程池中的线程会重新创建。为了避免在同一时间创建所有线程，该项配置表示在创建两个线程之间停顿时间，默认1000,如果没有配置，线程不会创建。

Connectors配置选项

HTTP Connector

简介

HTTP Connector 是一个支持HTTP/1.1协议的Connector组件。每一个HTTP Connector都是一个独立的web环境。用于接收http请求并返回响应结果。能够执行servlet和jsp页面。每一个请求都会有一个线程去处理，允许最大有多少个线程同时处理请求取决于maxThreads配置的数量。超过这个数量的请求会进入等待队列，进入等待队列的请求数取决于acceptCount
属性的配置项，如果超过这个数量，服务器就会返回拒绝连接的信息。

通用属性配置

属性 描述
allowTrace boolean值，该配置项用于配置http是否启用处理trace请求的方法，如果不需要，该项配置为false
asyncTimeout 异步请求处理的超时毫秒数，如果没有配置，则默认为servlet的超时时间30000,30秒
defaultSSLHostConfigName 用于安全连接的SSLHostConfig的默认的名称，如果客户端不支持SNI或者支持了但没有匹配的SSLHostConfig，就会使用默认的SSLHostConfig
enableLookups boolean值，是否禁用DNS反查，如果禁用了，调用request.getRemoteHost()将会返回ip地址，不查询域名
maxHeaderCount 容器中每个请求所允许的请求头的最大数量，超过这个数量将会被拒绝，小于0表示没有限制，默认值是100
maxParameterCount 每个get或者post请求的参数对的最大数量，小于0表示没有限制，默认值是10000，可以使用FailedRequestFilter过滤该限制
maxPostSize 每个post请求的参数体的数据量的最大值，该配置项可以限制那些参数体的数据量小于0的，如果没有配置，默认值是2MB(2097152)
maxSavePostSize
parseBodyMethods
port 服务器处理http请求时监听的端口号，操作系统只允许有一个应用监听同一个ip的同一个端口
protocol 接收请求所使用的通信协议，默认的是HTTP/1.1
proxyName 如果连接器使用了代理配置，调用request.getServerName()时，会返回这个配置项的值
proxyPort
redirectPort
scheme
secure
URIEncoding
useBodyEncodingForURI
useIPVHosts
xpoweredBy
